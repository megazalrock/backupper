# README Updater

## 概要

README.mdをプロジェクトの構造やインターフェースに合わせて更新するスキルです。

## 更新すべきタイミング

以下の変更があった場合にREADME.mdを更新してください：

- **CLIオプションの追加・変更・削除**: コマンドやフラグの変更
- **設定ファイル（config.ts）の構造変更**: 新しい設定項目の追加や既存項目の変更
- **新機能の追加**: ユーザーが利用できる新しい機能
- **インストール・セットアップ手順の変更**: 依存関係や初期設定の変更
- **技術スタックの変更**: 使用ライブラリやツールの変更

## README.mdの構造

現在のREADME.mdは以下のセクションで構成されています。更新時はこの構造を維持してください：

1. **機能** - ツールの主要機能の箇条書き
2. **インストール** - `bun install` コマンド
3. **セットアップ** - 初期設定の手順
4. **使用方法** - 基本的なコマンドの使い方
5. **CLIオプション** - 各コマンドのオプション一覧（テーブル形式）
6. **設定ファイル** - config.tsのサンプルと設定項目の説明
7. **テスト** - テストコマンド
8. **リンター** - リンターコマンド
9. **技術スタック** - 使用技術一覧
10. **ライセンス** - ライセンス情報

## 更新時の注意点

### CLAUDE.mdとの整合性

CLAUDE.mdにも類似の情報（コマンド、CLIオプション、構成など）が記載されています。README.mdを更新する際は、CLAUDE.mdとの整合性を確認してください。

- **CLAUDE.md**: 開発者・AI向けの詳細な技術情報（ディレクトリ構成、命名規則、import規約など）
- **README.md**: ユーザー向けの使用方法中心の情報

両者で重複する情報（CLIオプションなど）は、内容を一致させてください。

### テーブル形式

CLIオプションや設定項目はテーブル形式で記載してください：

```markdown
| オプション | 短縮形 | 説明 |
|-----------|--------|------|
| `--config <パス>` | `-c` | 設定ファイルのパスを指定 |
```

### コードブロック

- シェルコマンドは `bash` を指定
- 設定ファイルは `typescript` を指定

```markdown
\`\`\`bash
bun run backup
\`\`\`
```

### 日本語で記述

このプロジェクトのREADME.mdは日本語で記述してください。
